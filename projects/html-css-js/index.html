1) 10 most useful HTML tags (and why)

  1. <div> — generic container for grouping layout/content.
  2. <span> — inline wrapper for styling small text parts.
  3. <header> — top area of a page/section (semantic).
  4. <nav> — navigation links (semantic + accessibility).
  5. <main> — main content (improves structure).
  6. <section> — groups related content in sections.
  7. <article> — standalone content like a blog post.
  8. <h1..h6> — headings; important for structure + SEO.
  9. <a> — links (navigation + citations).
  10. <img> — images; use alt for accessibility/SEO.

Example: A blog post page is often <header> for the top bar, <main> for the post, <article> for the post content, <section> for related posts, and <footer> for credits/links.

Tip: Prefer semantic tags (header/nav/main/section/article/footer) because they help screen readers, search engines, and future you.

2) SEO: important “head” tags you should add

SEO is mostly about: (1) crawlability, (2) clear meaning of content, (3) good metadata for previews, and (4) performance + UX. These tags help search engines understand and display your page better.

  • <title> — appears in search results tab/title; keep it unique.
  • <meta name="description"> — influences the search snippet text.
  • <meta charset="utf-8"> — proper character encoding.
  • <meta name="viewport"> — required for mobile-friendly pages.
  • <link rel="canonical"> — avoids duplicate-content confusion.
  • <meta name="robots"> — tells bots index/follow rules.
  • Open Graph tags (og:title/og:description/og:image) — better link previews on social.
  • Structured data (JSON-LD script) — helps rich results (optional but powerful).

Also, inside the body, your heading order matters: Use one clear <h1> per page, then <h2> for sections, then <h3> for subsections. That hierarchy helps both humans and crawlers.

4B) Itemized list: same text, 6 font sizes (inline styling)

Same text, different units. Each item explains when you’d use that unit with an example.

  • Same text — 12px (absolute): use for pixel-perfect UI controls (e.g., tiny labels in a dashboard).
  • Same text — 16px (absolute): common base size; use when a design system specifies exact pixels.
  • Same text — 1em (relative to parent): use inside components so text scales if the component’s base size changes.
  • Same text — 1.25rem (relative to root): use for consistent typography scale across the whole site (headings/buttons).
  • Same text — 120% (relative): use when you want proportional sizing based on inherited size (e.g., make a callout slightly larger).
  • Same text — 3vw (viewport-based): use for responsive hero text that scales with screen width (careful: can get too small on phones).
